Pourquoi Xpansio

Données réelles. Analyse experte. Résultat actionnable.

01
Données entreprise réelles
OpenCorporates centralise les registres de commerces de 150+ pays. Xpansio interroge cette base pour récupérer les données légales réelles de votre entreprise en quelques secondes.
02
Actualités économiques du jour
NewsAPI + Reuters, BBC, Les Echos, Euronews — les dernières nouvelles du commerce international filtrées et injectées dans chaque analyse pour contextualiser chaque marché.
03
Analyse IA calibrée sur vos ressources
Claude interprète toutes ces données en tenant compte de votre taille, budget et équipe réels. Le score reflète ce qui est faisable pour vous — pas le potentiel théorique du marché.
04
Projections financières par pays
Investissement d'entrée estimé, CA potentiel an 1 et an 3, délai de rentabilité — pour comparer les opportunités et arbitrer entre marchés avec des chiffres concrets.
05
Aucune clé API exposée
Toutes les clés Anthropic, NewsAPI et OpenCorporates restent côté serveur. Votre navigateur ne communique qu'avec le backend Xpansio — jamais directement avec des APIs tierces.
06
Stratégie d'entrée sur mesure
Pour chaque marché positif : canal recommandé, type de partenariat, et une première action concrète à mener dès le premier mois.
Architecture

Comment Xpansio fonctionne

01
Recherche d'entreprise
Le frontend envoie le nom au backend → /api/company interroge OpenCorporates et retourne les données légales réelles.
02
Actualités en temps réel
→ /api/news interroge NewsAPI (ou RSS en fallback) et retourne les articles filtrés du commerce international.
03
Analyse par Claude
→ /api/analyze reçoit toutes les données, construit un prompt expert et appelle Claude — jamais depuis le navigateur.
04
Résultats classés
Score, verdict, projections financières et stratégie d'entrée — triés du plus au moins prioritaire.

Questions fréquentes

Tout ce que vous devez savoir

Xpansio interroge l'API publique OpenCorporates qui centralise les registres de commerces de 150+ pays. Vous obtenez le nom légal, le pays d'enregistrement, le statut, le type de société et la date de création. Cette requête se fait depuis le serveur backend, jamais depuis votre navigateur.
Oui. Les clés Anthropic, NewsAPI et OpenCorporates ne sont stockées que dans le fichier .env du serveur backend. Le frontend ne connaît que l'URL de votre propre backend — jamais les clés directement.
Les actualités sont récupérées depuis NewsAPI (ou les flux RSS Reuters, BBC, Les Echos, Euronews en fallback) à chaque chargement de la page. Les données d'entreprise d'OpenCorporates reflètent l'état des registres publics mis à jour régulièrement. Les analyses Claude sont générées en temps réel à chaque appel.
Jusqu'à 50 pays. L'analyse est automatiquement découpée en lots de 4 pays par appel Claude, ce qui évite les timeouts et garantit une réponse pour chaque lot. Un lot qui échoue est signalé sans bloquer les autres.
Le backend Node.js peut être déployé sur Railway, Render, Fly.io ou un VPS. Créez un fichier .env avec vos clés (ANTHROPIC_API_KEY, NEWS_API_KEY, OPENCORPORATES_API_KEY), lancez "npm install" puis "npm start". Le backend sert aussi le frontend statique.
